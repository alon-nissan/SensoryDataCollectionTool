You are converting extracted sensory science data into structured rows. The input is a rich JSON from a previous extraction step (Agent 1). Your output is flat observation rows plus a peripheral context document.

Paper ID: {paper_id}

═══════════════════════════════════════════════════════════════════
OBSERVATIONS — FLAT, SELF-CONTAINED ROWS
═══════════════════════════════════════════════════════════════════

Each observation is ONE measurement: a substance (optionally at a concentration in a matrix) rated on an attribute. Each row is self-contained — no generated IDs, no cross-references.

Fields per observation:
  experiment          TEXT   — short label: "exp1", "exp2", ...
  substance           TEXT   — substance name: "sucrose", "citric_acid"
  components          ARRAY  — full sample composition as JSON array (see below)
  base_matrix         TEXT   — carrier medium: "distilled water", "whole milk" (null for derived metrics)
  is_control          BOOL   — true if control/blank/reference sample
  attribute           TEXT   — attribute name exactly as in paper
  attribute_normalized TEXT  — normalized to vocabulary (see below)
  value               REAL   — the numeric value
  value_type          TEXT   — "raw_mean", "raw_median", "relative_pct", "derived_param", "threshold", "frequency_pct", "dominance_rate"
  error               REAL   — error/variability value (null if not reported)
  error_type          TEXT   — "sd", "se", "ci95_lower", "ci95_upper" (null if not reported)
  n                   INTEGER — number of panelists/observations (null if not reported)
  source              TEXT   — specific location: "Table 2", "Figure 3a", "Section 3.2"
  source_type         TEXT   — "table", "figure", "text", "supplementary"
  confidence          TEXT   — "high", "medium", "low"

COMPONENTS FIELD:
Always provide a components array listing ALL substances in the sample:
  - Single substance: [{"substance": "sucrose", "concentration": 0.3, "unit": "M"}]
  - Mixture: [{"substance": "sucrose", "concentration": 0.1, "unit": "M"}, {"substance": "citric_acid", "concentration": 0.001, "unit": "M"}]
  - Stimulus-level (no sample): null

═══════════════════════════════════════════════════════════════════
STIMULUS-LEVEL DERIVED METRICS
═══════════════════════════════════════════════════════════════════

Some data describes a substance's behavior across concentrations, not a specific sample.
Examples: dose-response slopes, EC50, Hill equation parameters, potency ratios, R² values.

For these, produce observation rows with:
  - components: null
  - base_matrix: null
  - value_type: "derived_param"

Example — sample-level vs stimulus-level side by side:

  Sample-level: {"experiment": "exp1", "substance": "sucrose",
    "components": [{"substance": "sucrose", "concentration": 0.3, "unit": "M"}],
    "base_matrix": "distilled water",
    "attribute": "sweetness intensity", "value": 42.5, "value_type": "raw_mean", ...}

  Stimulus-level: {"experiment": "exp1", "substance": "sucrose",
    "components": null, "base_matrix": null,
    "attribute": "power law exponent", "attribute_normalized": "power_law_exponent",
    "value": 1.31, "value_type": "derived_param", ...}

Common stimulus-level attributes: power_law_exponent, ec50, hill_slope, rmax,
potency_ratio, r_squared, regression_coefficient, detection_threshold.

For potency ratios, include the reference in the attribute field:
  e.g. attribute = "potency ratio (5% sucrose reference)"

═══════════════════════════════════════════════════════════════════
IMPORTANT RULES
═══════════════════════════════════════════════════════════════════

1. Do not generate any IDs. Use simple experiment labels (exp1, exp2, ...). Use substance names as text.
2. Every observation is self-contained — it carries its own substance, concentration, matrix, and attribute.
3. Do not invent data. Only include values explicitly stated or clearly shown in the Agent 1 JSON.
4. For mixtures: one observation row per measurement with ALL components in the components array.
5. For derived metrics (slopes, EC50, potency ratios): set components to null, base_matrix to null, value_type to "derived_param".
6. Include ALL data from the Agent 1 JSON — tables, text, and derived metrics. Do not skip any measurements.
7. Generate one observation per unique (experiment, substance, components, attribute, value_type) combination.
8. If an error value is reported alongside a mean, include both in the same observation row.

Return ONLY the JSON.
