═══════════════════════════════════════════════════════════════════
PAPER ID
═══════════════════════════════════════════════════════════════════

The paper_id for this paper is: {paper_id}

This ID is derived from the DOI by replacing all non-alphanumeric characters with underscores, collapsing consecutive underscores, and lowercasing.
Example: "10.1016/j.foodqual.2018.01.001" → "10_1016_j_foodqual_2018_01_001"

All IDs you generate must use this paper_id as a prefix.

═══════════════════════════════════════════════════════════════════
ID CONVENTIONS
═══════════════════════════════════════════════════════════════════

Use double-underscore (__) as the separator between the paper_id prefix and the entity suffix:

  experiment_id:  "{paper_id}__exp{N}"
                  e.g. "10_1016_j_foodqual_2018_01_001__exp1"

  stimulus_id:    "{paper_id}__{normalized_substance_name}_{N}"
                  e.g. "10_1016_j_foodqual_2018_01_001__sucrose_1"
                  Use the substance's normalized_name (lowercase, underscores, no spaces).

  sample_id:      "{paper_id}__sample_{N}"
                  e.g. "10_1016_j_foodqual_2018_01_001__sample_1"

N is a 1-based sequential integer. Keep numbering consistent and ordered.

═══════════════════════════════════════════════════════════════════
THREE-LEVEL HIERARCHY: SUBSTANCE → STIMULUS → SAMPLE
═══════════════════════════════════════════════════════════════════

Understand this hierarchy carefully:

1. SUBSTANCE (global chemical entity):
   A unique chemical compound that exists across all papers. Example: "sucrose".
   One row in the `substances` table. Has a globally unique normalized_name.

2. STIMULUS (paper-specific sourced instance):
   A particular sourced instance of a substance used in THIS paper.
   Example: "Sucrose from Sigma-Aldrich, >99% purity, crystalline powder".
   One row in the `stimuli` table. References a substance_id.
   Different papers using sucrose from different suppliers = different stimuli, same substance.

3. SAMPLE (what panelists actually tasted):
   A specific preparation presented to panelists, composed of one or more stimuli at specific concentrations dissolved/mixed in a base matrix.
   Example: "0.3 M sucrose in distilled water".
   One row in the `samples` table. Linked to stimuli via `sample_components`.

CRITICAL — SAMPLE COMPOSITION:
Every sample MUST have a base_matrix value. The base matrix is the carrier medium (e.g., "distilled water", "whole milk", "orange juice"). Each sample is composed of:
  1. One or more STIMULI (linked via the sample_components junction table) at specific concentrations
  2. A BASE MATRIX (stored directly on the sample row in the base_matrix column)
These are distinct concepts: stimuli are the test compounds; base matrix is what they are dissolved/mixed in. If the base matrix is unclear, check the experimental methods or conditions. Default to "water" only if the methods section clearly implies aqueous solutions without specifying the type. Never leave base_matrix empty — if truly unknown, set to "not reported".

═══════════════════════════════════════════════════════════════════
ENUM VALUES REFERENCE
═══════════════════════════════════════════════════════════════════

value_type (what kind of numeric value):
  - raw_mean: arithmetic mean of panelist ratings
  - raw_median: median of panelist ratings
  - relative_pct: percentage relative to a reference (e.g., sweetness as % of reference)
  - derived_param: derived parameter from model fitting (e.g., Stevens' exponent, Imax from TI)
  - threshold: detection or recognition threshold concentration
  - frequency_pct: percentage of panelists selecting an attribute (CATA, TCATA)
  - dominance_rate: dominance rate from TDS analysis

source_type (where the data came from):
  - table: extracted from a data table
  - figure: extracted from a figure (read off plot, estimated from graph)
  - text: extracted from inline text in the manuscript
  - supplementary: from supplementary materials

extraction_confidence (how confident you are in the value):
  - high: value is clearly stated, unambiguous
  - medium: value is stated but some interpretation needed (unit inference, table layout ambiguity)
  - low: value is approximate (read from figure, imprecise text description)

error_type (type of error/variability measure):
  - sd: standard deviation
  - se: standard error of the mean
  - ci95_lower: lower bound of 95% confidence interval
  - ci95_upper: upper bound of 95% confidence interval

citric acid, NaCl, caffeine, quinine, etc. should already be in the aliases dictionary.

DERIVED METRICS → RESULTS MAPPING
═══════════════════════════════════════════════════════════════════

Agent 1 may include a `derived_metrics` array within each experiment containing model
parameters (Stevens' law exponents, Hill equation parameters like Imax/K/n, EC50 values,
potency ratios, slopes, intercepts, R² values, etc.). Convert each derived metric into
a results row as follows:

  - metric_type → attribute_raw (e.g., "power_law_exponent", "potency_ratio_5pct_sucrose")
  - description → use to build a clear attribute_raw if metric_type alone is ambiguous
  - value → value
  - value_type = "derived_param"
  - se / ci_lower / ci_upper → error_value / error_type (map appropriately)
  - data_source → source_location
  - model / parameters / conditions → store as JSON string in context_json
  - extraction_confidence = "high" (these come from tables with exact values)
  - source_type = "table" or "text" (based on data_source)

For metrics that apply to a specific substance/sweetener (e.g., "slope for sucrose"),
match to the appropriate sample_id. If the metric is experiment-level and not
sample-specific, set sample_id to null.

For potency ratios, create one result row per substance × reference level combination:
  - attribute_raw = "potency_ratio" or the specific label from the paper
  - Store the reference conditions (e.g., "relative to 5% sucrose") in context_json
