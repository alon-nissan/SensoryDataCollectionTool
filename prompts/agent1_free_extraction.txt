You are extracting data from a sensory science research paper for a structured database. Your task is to produce a single, comprehensive JSON object that captures ALL reported information from the paper. Extract everything — the structuring and normalization will be handled by a downstream agent.

IMPORTANT PRINCIPLES:
- Extract EVERY reported number — means, SDs, SEs, confidence intervals, p-values, F-statistics, sample sizes, effect sizes, percentages.
- Always note data_source for every piece of data: "Table 1", "Figure 2A", "text p.5", "inline text in Results", "Materials and Methods section", etc.
- Look for multiple experiments (Experiment 1, Study A/B, Phase 1/2, Part I/II). Each gets its own entry in the experiments array.
- For dose-response data: each concentration is a separate stimulus entry.
- Capture ALL compounds in stimulus composition with their concentrations and units (M, mM, µM, g/L, % w/v, % w/w, ppm, mg/L, etc.).
- Include controls (water, blank, vehicle) and reference samples (e.g., standard solutions).
- Note any data available only in figures (not tables) — these will be extracted by a vision agent later. Still record what you can infer from figure captions and in-text references to figures.
- If information is not reported in the paper, omit the field rather than guessing. Do NOT fabricate values.
- Preserve the authors' original terminology for sensory attributes — do not rename or normalize them.

CONCEPTUAL HIERARCHY — SUBSTANCE → STIMULUS → SAMPLE:
Understanding this hierarchy will help you organize extracted data correctly.

1. SUBSTANCE = an abstract chemical entity (e.g., "sucrose", "citric acid", "capsaicin").
   Just name the compound clearly — you do not need to create any special structure for it.

2. STIMULUS = a paper-specific sourced instance of a substance, including supplier, purity, and form details.
   Example: "Sucrose (Sigma-Aldrich, ≥99%, crystalline powder)".
   Different papers may use the same substance from different suppliers — these are different stimuli.

3. SAMPLE = what panelists actually tasted. A specific preparation containing one or more stimuli at specific concentrations dissolved/mixed in a base matrix.
   Example: "0.3 M sucrose in distilled water" or "5% sucrose + 0.01 M citric acid in spring water".

Key rules:
- In dose-response studies, each concentration level is a SEPARATE SAMPLE (not a separate stimulus). The stimulus is the compound itself; the samples differ only in concentration.
  Example: Testing sucrose at 7 concentrations = 1 stimulus, 7 samples.
- Mixtures are SINGLE SAMPLES with multiple stimulus components. If panelists tasted "0.1 M sucrose + 0.001 M citric acid", that is one sample containing two stimuli at specific concentrations.
- Controls (water blanks, vehicle controls) are also samples — flag them with is_control: true.
- Capture supplier, purity, and form information at the stimulus level in the "stimuli" array, not repeated in each sensory_data entry.

CRITICAL — BASE MATRIX:
Every sample has a base matrix — the carrier medium in which stimuli are dissolved or mixed. Common base matrices include: distilled water, deionized water, spring water, whole milk, skim milk, orange juice, apple juice, yogurt, custard, model wine solution, gelatin gel, etc. Always extract and record the base matrix for each sample. If the base matrix is not explicitly stated, look for it in the Methods section (e.g., "all solutions were prepared in deionized water"). If truly not reported, note this in data_gaps.
A sample is ALWAYS composed of: (1) one or more stimuli at specific concentrations, AND (2) a base matrix. These are distinct concepts — do not merge them.

CONCENTRATION EXTRACTION (CRITICAL):
When concentration series appear in the main paper tables (e.g., Table 2 listing
concentrations in % w/v, M, mM, ppm, etc.), ALWAYS extract them as part of the
stimulus/sample composition — even if supplementary data exists with different units.
Record both the main-text value and any supplementary reference. Never leave
concentration as null when the paper provides it in any form. If a concentration
appears in one unit in the main text and another unit in the supplement, extract BOTH
with their respective sources noted.

For dose-response studies, concentrations are the x-axis variable — they are essential
for data interpretation. Each concentration level should become a separate sample with
its concentration value explicitly recorded.

ADDITIONAL GUIDELINES:

1. SENSORY DATA FLEXIBILITY: The sensory_data array should adapt to whatever data the paper presents. This could be:
   - Dose-response curves (intensity vs. concentration)
   - Attribute profiles (multiple attributes per stimulus)
   - CATA check-all-that-apply frequencies
   - TDS temporal dominance rates at time points
   - Hedonic ratings (liking/preference scores)
   - Threshold values
   - Ranking data
   - Time-intensity curve parameters
   Create as many data blocks as needed to capture all reported data.

2. MULTIPLE DATA SOURCES: A single result may appear in multiple places (e.g., a mean in both a table and a figure). Record it once with the most precise source (prefer tables over figures, as tables give exact numbers).

3. STIMULUS COMPOSITION: Be thorough. If a paper tests "0.1 M sucrose + 0.001 M citric acid", the composition object should have entries for both compounds with their concentrations and units.

4. STATISTICAL DETAIL: For ANOVA tables, extract every row. For post-hoc tests, record all pairwise comparisons and their significance levels if reported.

5. FIGURES: Even without vision extraction, you can often infer what data a figure contains from its caption, axis labels mentioned in text, and in-text references. Record this in figure_inventory with data_extracted: false.

6. FIGURE RELEVANCE SCORING: For each figure in figure_inventory, assign a relevance_score
   between 0.0 and 1.0 indicating how likely the figure contains extractable sensory data
   points (means, error bars, frequencies, etc.):
   - 0.8-1.0: Almost certainly contains extractable sensory data (bar charts of intensity
     ratings, dose-response curves, line graphs of sensory scores, heatmaps of sensory profiles)
   - 0.5-0.7: Likely contains some data but may be partially redundant with tables or hard
     to read (spider/radar plots, box plots, small multiples)
   - 0.2-0.4: Unlikely to contain useful data (PCA biplots, MDS plots, dendrograms, flow
     charts) but may have some readable values
   - 0.0-0.1: No extractable sensory data (photographs, schematic diagrams, PRISMA charts,
     molecular structures, experimental setup illustrations)

   Figures whose data already appears in tables should receive lower scores.

   IMPORTANT — FIGURE-PRIMARY PAPERS: If the paper's primary quantitative results
   (e.g., dose-response data, mean sensory ratings) appear ONLY in figures and are NOT
   tabulated, score ALL data-bearing figures at 0.8+ regardless of overlap with other
   figures. In such papers, figures are the primary data source, not supplementary
   illustrations. Set "figure_primary_data": true in the study_metadata if this applies.
